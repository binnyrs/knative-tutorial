Eventing

Overview

Knative Eventing allows us to define and deploy Event Sources that can listen to external events, it can be imagined like a Producer-Consumer model. The services that are deployed via Knative Serving can then be able to consume the events or subset of events from the Producer via Feeds. All the messages that are exchanged follows the CloudEvents Specification.

The Eventing has three main building blocks:

  • Buses

  • Sources

  • Flows

The Eventing specification is currently under extensive development so expect lot of changes

Install Knative Eventing

Its assumed that you have already installed Knative Serving, if not please visit the section Install Knative Serving to set up the same.

kubectl apply --filename https://storage.googleapis.com/knative-releases/eventing/latest/release.yaml
