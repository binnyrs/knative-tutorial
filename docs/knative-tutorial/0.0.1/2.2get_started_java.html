Getting Started - Java

Overview

This section shows how to build and deploy an simple greeter Spring Boot Java application as Knative service

Prerequisites

The Spring Boot application skeleton will be generated using spring-init cli, you can download and install it form Spring CLI.

After installation ensure that the following command works correctly:

spring --version

Create Service

Create app directory

mkdir -p $PROJECT_HOME/build/getstarted/java
cd $PROJECT_HOME/build/getstarted/java

Create Spring Boot application

The following command creates an empty Spring Boot application in the directory $PROJECT_HOME/build/getstarted/java

spring init ./

Update pom.xml

Update the pom.xml with the following content:

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
	xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
	<modelVersion>4.0.0</modelVersion>

	<groupId>com.example</groupId>
	<artifactId>greeter</artifactId>
	<version>0.0.1-SNAPSHOT</version>
	<packaging>jar</packaging>

	<name>greeter</name>
	<description>Spring Boot Greeter</description>

    <properties>
		<project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
		<project.reporting.outputEncoding>UTF-8</project.reporting.outputEncoding>
        <java.version>1.8</java.version>
        <spring-boot-version>2.0.5.RELEASE</spring-boot-version>
    </properties>

  <dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-dependencies</artifactId>
            <version>${spring-boot-version}</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
  </dependencyManagement>

	<dependencies>
		<dependency>
			<groupId>org.springframework.boot</groupId>
			<artifactId>spring-boot-starter</artifactId>
        </dependency>

        <dependency>
			<groupId>org.springframework.boot</groupId>
			<artifactId>spring-boot-starter-web</artifactId>
		</dependency>

		<dependency>
			<groupId>org.springframework.boot</groupId>
			<artifactId>spring-boot-starter-test</artifactId>
			<scope>test</scope>
		</dependency>
	</dependencies>

    <build>
        <finalName>${artifactId}</finalName>
		<plugins>
			<plugin>
				<groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
                <version>${spring-boot-version}</version>
                <executions>
					<execution>
						<goals>
							<goal>repackage</goal>
						</goals>
					</execution>
				</executions>
			</plugin>
		</plugins>
	</build>


</project>

Create Greeter Controller

Create a new file called GreeterController.java in the folder $PROJECT_HOME/build/getstarted/java/com/example/demo with the following content:

package com.example.demo;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

/**
 * GreeterController
 */
@RestController
public class GreeterController {

    @Value("${MESSAGE:Knative on OpenShift}")
    private String message;

    @GetMapping("/")
    public String greet() {
        return"Java::"+message;
    }
}

Create a Dockerfile

FROM fabric8/java-jboss-openjdk8-jdk:1.3.1
ENV JAVA_APP_DIR=/deployments
ENV AB_OFF=true
EXPOSE 8080
COPY target/greeter.jar /deployments/

Build Service

From $PROJECT_HOME/build/getstarted/java:

Local

The application can be built using maven, run the following command to build and start the application locally,

./mvnw -DskipTests clean package
./mvnw spring-boot:run (1)
1 This will start the application locally and accessible via http://localhost:8080/. Use Ctrl+C to terminate the process.

OpenShift

Docker Image

docker build --rm -t dev.local/greeter:2.0.0 .

Deploy Service

To deploy the Knative service we need to create file called service.yaml in $PROJECT_HOME/build/getstarted/java with the following content:

Create service.yaml

apiVersion: serving.knative.dev/v1alpha1
kind: Service
metadata:
  name: greeter
spec:
  runLatest:
    configuration:
      revisionTemplate:
        spec:
          container:
            image: dev.local/greeter:2.0.0 (1)
1 It is very important that the image is a fully qualified name docker image name with tag.

Deploy Service

kubectl apply -f service.yaml

After successful deployment of the service we should see a Kubernetes deployment called greeter-00002-deployment available in the OpenShift dashboard:

Invoke Service

The applications can be invoked only using knative-ingress-gateway which was deployed with Knative serving deployment.

The following snippet shows how to invoke the greeter nodejs service that was deployed earlier:

export IP_ADDRESS=$(kubectl get node -o 'jsonpath={.items[0].status.addresses[0].address}')

export SERVICE_NODEPORT=$(oc get svc knative-ingressgateway -n istio-system -o 'jsonpath={.spec.ports[?(@.port==80)].nodePort}')

export HOST_URL=$(kubectl get routes.serving.knative.dev greeter  -o jsonpath='{.status.domain}')

curl -H "Host: ${HOST_URL}" ${IP_ADDRESS}:${SERVICE_NODEPORT}

The curl command above should return a response like Java::Knative on OpenShift
